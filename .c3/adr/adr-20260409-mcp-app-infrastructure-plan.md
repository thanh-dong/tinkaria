## Goal

Provision MCP Apps as a documented architectural base centered on jobs-to-be-done before implementation. The server side should own app manifests, registry, lifecycle, and policy; the client should host a shared embed shell; and the existing transcript artifact model should remain the fallback path. Important jobs must be completed correctly before delight or richer app polish expands the scope.
