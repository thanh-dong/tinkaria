## Goal

Render transcript message surfaces and grouping boundaries for assistant responses, WIP narration/tool work, dedicated special-tool interactions, present_content artifacts, system/user/result/status messages, compact summaries, and rich artifact cards.

### Tool-Specific Rendering in ToolCallMessage

ToolCallMessage switches on tool type for specialized expanded content:

| Tool | Expanded View |
| --- | --- |
| Edit | FileContentView with unified diff (oldString vs newString) |
| Read | ImageContentView (if images) or FileContentView |
| Write | FileContentView showing written content |
| Bash | Input command + result, with timeout/background flags |
| Others | Input code block + result/error |
Tool labels generated by `getToolLabel()`: file paths for file ops, query text for search, skill names for skills, MCP format for MCP tools.

### Dedicated Tool Interaction Patterns

**AskUserQuestionMessage** — Three states:

1. Active (isLatest): Multi-question stepper with progress bar, single/multi-select, custom Other input, auto-advance on single-select
2. Pending (not latest): Grayed out newer question active message
3. Completed: Definition list showing Q&A pairs
**ExitPlanModeMessage** — Three states:

1. Active: Plan markdown + three action buttons (Approve & Clear, Approve, Suggest Edits)
2. Pending: Grayed out
3. Complete: Shows user action taken (Approved, Adjusted, Discarded)
**TodoWriteMessage** — Live progress list with status-based icons: Check for completed, animated Loader2 for in_progress, Circle for pending.

### Shared Primitives (shared.tsx)

Layout: MetaRow, MetaContent, MetaSeparator, MetaLabel, MetaText, ExpandableRow, MetaCodeBlock, MetaPill, VerticalLineContainer.

Markdown: `createMarkdownComponents(options)` overrides all HTML tags with project-themed styles, handles local file links via OpenLocalLinkProvider context, auto-detects embed languages in code fences.

Context: `TranscriptActionsContext` provides onRetryChat, onNewChat, onResumeSession, onDismissError callbacks.

## Container Connection

Part of c3-1 (client). ChatTranscript maps hydrated messages into render items, then message renderers own visible surfaces: WipBlock for progress/tool work, TextMessage for final or live assistant response text, PresentContentMessage for structured artifacts, and rich-content primitives for markdown/code/embed overlays. Assistant text may also auto-upgrade known embeddable links such as Diashort into rich artifact cards when the model emits a plain link instead of a dedicated present_content tool call.
