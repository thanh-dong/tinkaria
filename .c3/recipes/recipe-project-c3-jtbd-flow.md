## Goal

1. User opens a project and gets immediate orientation: overview, topology, and key ownership cues.
2. User drills into a component or area to answer lookup and impact questions.
3. User continues into chat or transcript work with the same architectural context available.
4. If interactive render is blocked or unsupported, the user still receives the same answer path via `present_content` artifacts.
